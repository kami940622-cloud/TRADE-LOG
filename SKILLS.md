# TRADE LOG — Claude 스킬 사용 가이드

> 스킬은 서브에이전트(별도 Claude 인스턴스)를 띄워서 실행되므로
> 직접 처리보다 **메인 컨텍스트 소모가 적습니다.**
> 무거운 작업일수록 스킬로 위임하는 게 유리합니다.

---

## 🟠 적극 추천

### `/tdd`
**언제:** PNL/ROI 계산, LONG/SHORT 분리, 가중평균 진입가 등 **계산 로직을 수정할 때**

**사용법:**
```
/tdd PNL 계산 로직 수정할 건데 테스트 먼저 짜줘
```
```
/tdd SHORT 손절 판정 함수 검증해줘
```

**이유:** 계산 버그가 반복적으로 발생했던 프로젝트. 테스트가 있으면
다음 수정 시 안전망이 생깁니다.

---

### `/code-review`
**언제:** 기능 추가 후 **코드 품질/버그 검토**가 필요할 때

**사용법:**
```
/code-review
```
```
/code-review --fix   ← 발견한 문제 자동 수정까지
```

**이유:** diff 기준으로만 검토해서 컨텍스트를 아낍니다.
Phase 2(PLAN), Phase 3(피드백) 추가 후 꼭 한 번 돌리세요.

---

### `/simplify`
**언제:** 파일이 3000줄 넘어가거나 **기능 추가 후 코드가 비대해졌다 싶을 때**

**사용법:**
```
/simplify
```
또는 그냥 자연어로:
```
코드 좀 정리해줘 / 중복 있는지 봐줘
```

**이유:** 단일 HTML 파일 구조라 기능이 쌓일수록 비대해집니다.
주기: Phase 2/3 추가 완료 시점마다 한 번씩 권장.

---

## 🟡 상황에 따라 유용

### `/diagnose`
**언제:** "진행중 2인데 없다" 같은 **데이터 불일치·버그** 발생 시

**사용법:**
```
/diagnose 진행중 카운트가 실제랑 다르게 나와
```
```
/diagnose 청산 저장하면 PNL이 이상하게 계산돼
```

**이유:** 재현 → 원인 → 수정 → 검증 순서로 체계적으로 접근합니다.
"일단 이렇게 해보세요" 식 추측 없이 근본 원인부터 파고듭니다.

---

### `/prototype`
**언제:** Phase 2 PLAN 기능 UI를 어떻게 짤지 **방향이 안 잡힐 때**

**사용법:**
```
/prototype PLAN 탭 UI 몇 가지 시안 만들어줘
```
```
/prototype 진행중 거래 카드 레이아웃 다르게 해보고 싶어
```

**이유:** 코드에 바로 손대기 전에 throwaway 시안을 빠르게 만들어
방향 확정 후 본 작업에 들어갈 수 있습니다.

---

### `/verify`
**언제:** 배포 후 **실제 브라우저에서 동작 확인**이 필요할 때

**사용법:**
```
/verify 방금 배포한 거 아이패드에서 하단 공백 없는지 확인해줘
```
```
/verify CALC → NEW 적용 버튼 제대로 동작하나 확인해줘
```

**이유:** CSS/레이아웃 수정 후 실제 화면과 코드가 맞는지
브라우저를 직접 열어서 확인합니다.

---

## 🔵 나중에 (데이터 쌓이면)

### `/to-prd`
**언제:** Phase 3 피드백 기능 본격 개발 전 **요구사항 정리**

**사용법:**
```
/to-prd Phase 3 피드백 기능 PRD 만들어줘
```

**이유:** 채찍질 항목(손절 미이행, 복수매매, 즉흥 vs 계획 승률 등)이
많아서 개발 전에 문서로 정리해두면 구현이 훨씬 명확해집니다.

---

## 💡 사용 팁

| 상황 | 추천 |
|------|------|
| 계산 로직 수정 | `/tdd` 먼저 → 수정 → 테스트 통과 확인 |
| 기능 추가 완료 | `/code-review` 로 마무리 |
| 파일 3000줄 초과 | `/simplify` |
| 버그 재현 안 됨 | `/diagnose` |
| UI 방향 못 잡겠음 | `/prototype` |
| 배포 후 확인 | `/verify` |

> 굳이 `/스킬명` 을 외울 필요 없이 자연어로 "검토해줘", "정리해줘",
> "버그 파줘" 라고 해도 Claude가 알아서 적절한 스킬을 씁니다.
> 단, 명시적으로 부르면 **컨텍스트 절약 효과**가 더 큽니다.
