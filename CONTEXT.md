# TRADE LOG — 도메인 용어 정의

## Trade (거래)
실제로 진입한 매매 기록. `result`는 `ACTIVE | WIN | LOSE`.
- `ACTIVE`: 청산 전 진행중
- `WIN`: 청산 완료, 수익
- `LOSE`: 청산 완료, 손실

## Plan (시나리오)
실제 진입 여부와 무관하게 **미리 잡아둔 매매 시나리오**.
"내가 계획한 시나리오가 맞았는가"를 추적하는 것이 핵심 목적.

### Plan.status
| 값 | 의미 |
|----|------|
| `PENDING` | 아직 가격이 진입 예정가에 닿지 않음 |
| `TRIGGERED` | 가격이 진입 예정가에 닿음 |
| `ENTERED` | 실제로 진입함 (Trade와 연결됨) |
| `MISSED` | 가격은 닿았으나 진입 못 함 (외출 등) |
| `CANCELLED` | 시나리오 자체를 폐기 |

## Session (세션)
진입 시각(KST) 기준 자동 분류되는 시장 세션.
- 아시아장: 09:00–18:00
- 런던장: 17:00–02:00
- 뉴욕장: 22:00–07:00
- 나스닥개장: 23:30 ±30분
- 나스닥마감: 06:00 ±30분

## Partial Entry (분할진입)
하나의 Trade에서 여러 번에 나눠 진입하는 것. 평균 진입가는 가중평균으로 자동계산.

## Partial Exit (분할청산)
하나의 Trade에서 여러 번에 나눠 청산하는 것. 유형은 `TP`(익절) 또는 `SL`(손절).

## PNL
실현 손익 (USDT 기준).
- LONG: `(청산가 - 진입가) × 수량`
- SHORT: `(진입가 - 청산가) × 수량`

## ROI
수익률 (%).
- LONG: `(청산가 - 진입가) / 진입가 × 100`
- SHORT: `(진입가 - 청산가) / 진입가 × 100`
